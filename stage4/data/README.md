# Stage 4 Data Requirements

Do not commit large raster files, source tables with restricted access, patch
catalogues, or model-ready arrays to GitHub.

Required data products are generated by previous thesis stages:

1. Stage 2 monthly bivariate class rasters.
2. Stage 3 monthly mean XCH4 rasters.
3. Stage 3 monthly 3deg and 5deg anomaly rasters.
4. Stage 3 monthly 3deg and 5deg continuous standardized-enhancement rasters.
5. Carbon Mapper plume/source CSVs containing at least:
   - `point_source_name`
   - `scene_timestamp`
   - gas or plume type field
   - source or plume coordinates

Data management requirements:

- keep rasters and source tables in Google Drive or a data repository;
- document exact folder paths, checksums, or DOI;
- retain only documentation and small configuration examples in GitHub.
